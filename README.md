# AuditSample Pro — Інструмент аудиторської вибірки (МСА 530)

**AuditSample Pro** — це веб-застосунок для професійних аудиторів, призначений для побудови репрезентативних вибірок, аналізу даних та екстраполяції помилок згідно з Міжнародними стандартами аудиту (МСА 530 "Аудиторська вибірка").

## 1. Логіка обробки генеральної сукупності

Перед застосуванням будь-якого методу вибірки програма виконує попередню обробку даних:

### А. Явно незначні суми (CTT - Clearly Trivial Threshold)
Згідно з МСА 450, аудитор може визначити поріг, нижче якого викривлення вважаються явно незначними.
- Елементи, сума яких менша за CTT, **виключаються** з генеральної сукупності, якщо їх загальна сума не перевищує допустиме викривлення (PM).
- Це дозволяє зменшити "шум" та зосередитись на суттєвих елементах.

### Б. Виділення ключових елементів (Key Items)
Програма автоматично відокремлює елементи, які мають перевірятися суцільним методом (100% покриття). Вони не підлягають вибірці, а помилки в них додаються до результату за фактичною величиною.

**Критерії відбору ключових елементів:**
1. **Суттєвість:** Елементи, вартість яких перевищує або дорівнює **Допустимому викривленню (PM)**.
2. **Статистичні аномалії (Modified Z-Score):**
   - Використовується робастний метод на основі Медіани та MAD (Median Absolute Deviation).
   - Формула: `Mi = 0.6745 * (Xi - Median) / MAD`.
   - Елемент вважається аномальним, якщо `|Mi| > 3.5`.
   - *Перевага:* На відміну від стандартного відхилення, цей метод не спотворюється наявністю екстремальних викидів у даних.

## 2. Методи вибірки та їх призначення

### 1. Монетарна вибірка (Monetary Unit Sampling - MUS)
Також відома як PPS (Probability Proportional to Size).
*   **Призначення:** Детальні тести активів (дебіторська заборгованість, ОЗ, запаси) та доходів. Найкраще виявляє **завищення** (Overstatement).
*   **Логіка:** Кожна грошова одиниця (напр., 1 грн) є одиницею відбору. Чим більша сума транзакції, тим вища ймовірність її потрапляння у вибірку.
*   **Формула розміру:** `n = (Вартість сукупності * Коеф. Надійності) / Допустиме викривлення`.
*   **Екстраполяція (Tainting):**
    - Розраховується частка викривлення (Tainting) для кожного елемента: `T = (Облікова - Аудит) / Облікова`.
    - Прогнозована помилка = `Sum(Tainting) * Інтервал вибірки`.

### 2. Випадкова вибірка (Simple Random Sampling)
*   **Призначення:** Універсальний метод для однорідних сукупностей або коли ризик не корелює з сумою (наприклад, перевірка наявності первинних документів).
*   **Логіка:** Кожен елемент має однакову ймовірність (1/N) бути обраним. Використовується генератор псевдовипадкових чисел (Mulberry32) з "зерном" (Seed) для відтворюваності.
*   **Формула розміру:** Статистична формула для оцінки середнього/пропорцій на основі Z-оцінки (рівень довіри), очікуваної помилки та точності.
*   **Екстраполяція:** Метод різниць (Difference Estimation). Середня помилка у вибірці множиться на кількість елементів у сукупності.

### 3. Атрибутивна вибірка (Attribute Sampling)
*   **Призначення:** Виключно для **Тестів контролів** (Test of Controls). Оцінює частоту відхилень (так/ні), а не грошову суму помилки.
*   **Логіка:** Розмір вибірки визначається за таблицями AICPA на основі трьох параметрів:
    1. Ризик надмірної довіри (Risk of Over-Reliance).
    2. Допустимий рівень відхилень (TDR).
    3. Очікуваний рівень відхилень (EDR).
*   **Екстраполяція:** Розраховується Верхня межа відхилень (CUER - Upper Deviation Limit) з урахуванням знайдених помилок.

### 4. Stop-or-Go (Послідовна вибірка)
*   **Призначення:** Оптимізація обсягу роботи, коли аудитор очікує **нуль помилок**.
*   **Логіка:**
    - Етап 1: Береться мінімальна вибірка (напр., 25 од.). Якщо помилок 0 — тест пройдено.
    - Етап 2: Якщо знайдено хоч одну помилку, вибірка розширюється (ще +25 од.) для уточнення результату.

### 5. Класична змінна вибірка (CVS)
*   **Призначення:** Перевірка на **заниження** (Understatement), наприклад, кредиторської заборгованості, або робота з даними, що містять нульові/від'ємні значення.
*   **Логіка:** Базується на нормальному розподілі та стандартному відхиленні (σ).
*   **Екстраполяція:** Difference Estimation або Mean-per-Unit. Будується довірчий інтервал навколо середнього значення.

### 6. Кластерна вибірка (Cluster Sampling)
*   **Призначення:** Аналітичні процедури та розвідувальний аналіз структури даних.
*   **Логіка:** Використовує алгоритм машинного навчання **K-Means++** для групування даних на кластери за вартістю.
*   **Відбір:** З кожного кластера автоматично відбираються:
    - Центроїд (типовий представник).
    - Мінімум та Максимум (граничні значення).

### 7. Аналіз Бенфорда (Benford's Law)
*   **Призначення:** Виявлення ознак шахрайства або маніпуляцій даними.
*   **Логіка:** Аналізує частоту появи першої цифри (1-9) у сумах та порівнює з еталонним розподілом (`log10(1 + 1/d)`).
*   **Результат:** Підсвічує цифри, частота яких статистично значуще відхиляється (Z-score > 1.96).

### 8. Аналіз Парето (ABC)
*   **Призначення:** Концентрація зусиль на найбільш суттєвих транзакціях.
*   **Логіка:** Відбирає топ-елементи, які сумарно формують 80% грошової вартості сукупності (принцип 80/20).

### 9. Процентильний відбір та Тест Граббса
*   **Percentile:** Відбирає Топ-N% найбільших та Топ-N% найменших значень (аналіз "хвостів" розподілу).
*   **Grubbs:** Статистичний тест для виявлення математичних викидів (outliers) у вибірці.

## 3. Екстраполяція результатів

Після внесення аудитором фактичних ("Аудиторських") значень програма автоматично розраховує:

1. **Прогнозоване викривлення (Projected Misstatement):** Найбільш ймовірна сума помилки у всій генеральній сукупності. Розраховується методом, специфічним для обраної стратегії (Tainting для MUS, Average Difference для Random/CVS).
2. **Верхня межа викривлення (Upper Misstatement Bound):** Прогнозована помилка + Резерв на ризик вибірки (Precision).
   - Якщо Верхня межа < Допустимого викривлення (PM) -> Вибірка приймається.
   - Якщо Верхня межа > PM -> Потрібне додаткове дослідження або коригування.

Помилки, знайдені у **Ключових елементах** (Key Items), додаються до результату за їх фактичною величиною і не екстраполюються, оскільки ці елементи перевіряються суцільно.

*   **Безпека:** Всі розрахунки виконуються виключно у браузері клієнта (Client-side). Дані не передаються на сервер.
*   **Відтворюваність:** Використовується генератор псевдовипадкових чисел `Mulberry32` з можливістю задати "зерно" (Seed). Це дозволяє аудитору отримати ідентичну вибірку при повторній перевірці.
*   **Експорт:** Підтримується експорт робочих документів у Excel (.xlsx) із збереженням формул та форматуванням, а також у JSON.
